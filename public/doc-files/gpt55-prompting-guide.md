# 不是模型降智，变笨了，而是懒得美化你的垃圾提示词

原文档链接：

- [GPT-5.5 Prompting Guide](https://developers.openai.com/api/docs/guides/prompt-guidance?model=gpt-5.5)
- [Using GPT-5.5](https://developers.openai.com/api/docs/guides/latest-model)

这篇不是玄学提示词教程，而是把 OpenAI 官方 GPT-5.5 prompting guide 翻成人话：

GPT-5.5 并不是更喜欢听你把每一步都念一遍。它更喜欢你讲清楚结果、边界、证据和交付格式，然后让它自己选择更高效的执行路径。

## 一句话结论

如果你感觉模型“变笨了”，先别急着骂模型。很多时候是因为你的提示词还停留在旧模型时代：又长、又绕、又喜欢把流程写死，却没说清楚什么叫做好。

GPT-5.5 的官方建议很明确：用更短、更结果导向的提示词，少搬运旧的 prompt stack。你需要告诉模型“我要什么结果”，而不是把它当成一个只能照抄步骤的实习生。

## GPT-5.5 更吃哪种提示词

好提示词要优先写这几件事：

- 目标结果：最后要交付什么。
- 成功标准：做到什么才算完成。
- 约束边界：哪些不能碰，哪些必须保留。
- 证据规则：哪些结论必须查资料、引用来源或验证。
- 输出格式：最终回答是清单、表格、JSON、代码补丁，还是简短结论。
- 停止规则：什么时候继续查，什么时候该停，什么时候必须问用户。

官方文档的核心意思很直白：GPT-5.5 更擅长从清晰目标出发，保留约束，并把产品意图变成具体动作。你越把“完成态”描述清楚，它越不需要靠猜。

## 别再写这种垃圾提示词

坏示例：

```text
你是世界上最强的 AI，（什么工程师，产品经理，软件巴拉巴拉）请认真思考，详细分析，一步一步来。
先阅读所有内容，再总结，再推理，再优化，再输出。
要求专业、高级、准确、完整、有深度。
```

这类提示词的问题是：看起来很努力，其实没给任何可执行标准。

- “认真”是什么意思？
- “高级”是什么风格？
- “完整”覆盖哪些内容？
- “准确”要不要引用来源？
- “一步一步”是否真的必须展示给用户？

模型不是读心术接口。你不给目标和约束，它只能用默认审美帮你补，而 GPT-5.5 的默认风格本来就更直接、更任务导向。

## 改成这样（我怎么写出这篇文档的）

好示例：

```text
目标：把官方 GPT-5.5 prompting guide 整理成一篇中文教程，读者是正在写 AI Agent 提示词的开发者。

成功标准：
- 先解释 GPT-5.5 为什么更适合 outcome-first prompt
- 给出坏提示词和改写后的好提示词
- 必须包含官方原文档链接
- 不要过度营销，不要写玄学表达

约束：
- 所有官方结论只基于 OpenAI 官方文档
- 不编造 API 参数、价格或发布日期
- 如果内容来自推断，要明确说是推断

输出：
- Markdown 文档
- 中文
- 标题、短段落、清单、代码块
```

这就是 GPT-5.5 更喜欢的结构：目标清楚，验收清楚，边界清楚，输出清楚。

## Personality 不是废话

GPT-5.5 默认更简洁、更直接、更任务导向。这个默认值适合生产系统，但如果你在做客服、教学、陪伴、咨询、销售、社区运营，就要明确写 personality 和 collaboration style。

不要写一大篇人格小说。写短一点：

```text
# Personality
你是一个耐心、直接、可信的技术助手。默认用中文回答，语气友好但不油腻。

# Collaboration style
当需求足够明确时直接推进；只有缺失信息会明显改变答案或带来风险时，才提出一个窄问题。
遇到不确定信息，先标注不确定性，再说明你会如何验证。
```

Personality 管“听起来像谁”，collaboration style 管“怎么一起做事”。它们不能替代目标、验收标准、工具规则和停止条件。

## Reasoning effort 不要无脑拉满

GPT-5.5 的 `reasoning.effort` 默认是 `medium`。官方建议把 `medium` 当成质量、可靠性、延迟和成本之间的平衡起点。

实用建议：

- 普通问答、改写、总结：先用 `low` 或默认。
- 多步骤任务、工具调用、代码修改：从 `medium` 开始。
- 长任务、复杂推理、严肃验证：再考虑 `high`。
- 最难的异步 Agent 或评测任务：才考虑 `xhigh`。

提示词写烂了，盲目拉高 reasoning effort 只是在花更多 token 给模型擦屁股。

## 工具调用要写规则，不要靠感觉

如果你的 Agent 要搜索、读文件、查数据库、跑测试、调用外部 API，就要明确工具规则：

```text
工具规则：
- 当工具能明显提升准确性、完整性或证据质量时，使用工具。
- 不要因为答案看起来明显就跳过必要查询。
- 如果查询结果为空或不完整，换一种检索方式重试。
- 只有任务完成且验证通过后，才输出最终结论。
```

GPT-5.5 的工具使用更强，但强不等于你可以不写边界。尤其是涉及事实、账单、代码、部署、法律、医疗、金融时，工具规则就是安全带。

## 推荐模板

直接拿去改：

```text
Role: [1-2 句说明模型的职责、上下文和任务]

# Personality
[语气、风格、专业度、是否需要幽默或共情]

# Goal
[用户最终要得到什么]

# Success criteria
[完成前必须满足的条件]

# Constraints
[业务规则、安全规则、证据要求、不可修改范围]

# Tools
[什么时候用工具、工具失败怎么办、哪些动作有副作用]

# Output
[输出语言、结构、长度、格式]

# Stop rules
[什么时候继续、什么时候停止、什么时候询问用户]
```

## 最后的提醒

GPT-5.5 不是不会美化你的提示词，而是它更不愿意被旧时代的废话 prompt 牵着走。你写清楚目标，它会更聪明；你只写“认真一点、高级一点、完整一点”，它就只能帮你猜。

别把提示词写成祈祷文。把它写成验收单。
